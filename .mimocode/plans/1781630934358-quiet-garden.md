# Fix: Icon build error in app.json

## Problem
`zepp build` throws: "The icon in app.json is empty or the image does not exist"

## Analysis
- `app.json` sets `"icon": "icon.png"` (line 11)
- Valid 73KB PNG files exist at:
  - `assets/gt.r/image/icon.png`
  - `assets/gt.s/image/icon.png`
- The build tool (`@zeppos/zpm`) cannot find them at the expected location

## Root Cause
The Zepp OS build tool resolves the icon path relative to a `common` assets directory first, then platform-specific. When no `assets/common/image/icon.png` exists, the tool fails.

## Fix
Copy `icon.png` into a shared `assets/common/image/` directory:

1. Create `assets/common/image/` directory
2. Copy `assets/gt.r/image/icon.png` to `assets/common/image/icon.png`

## Files to modify
- **Create**: `assets/common/image/icon.png` (copy from `assets/gt.r/image/icon.png`)

## Verification
Run `zepp build` and confirm it completes without the icon error.
